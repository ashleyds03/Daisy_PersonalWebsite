01 · Introduzione

Obiettivi del progetto

Il progetto descrive la progettazione e configurazione di una rete aziendale distribuita su quattro sedi: la sede centrale (HQ) a Milano, due filiali a Roma e Napoli, e un Data Center dedicato ai servizi centralizzati. La simulazione è realizzata in Cisco Packet Tracer.

  • ✦ Segmentare il traffico tramite VLAN per separare uffici, magazzino e direzione
  • ✦ Garantire la raggiungibilità tra tutte le sedi tramite routing dinamico RIP v2
  • ✦ Centralizzare i servizi DHCP e DNS nel Data Center
  • ✦ Implementare una DMZ fisica per isolare il server Web
  • ✦ Proteggere il perimetro con ACL firewall sulle interfacce WAN

04 · Routing dinamico

RIP versione 2

RIP v2 propaga dinamicamente le rotte tra tutti i router della rete. La configurazione usa no auto-summary per supportare VLSM e subnet mask classless.

Configurazione RIP v2 · Router HQ
router rip version 2 no auto-summary network 192.168.10.0 network 192.168.20.0 network 192.168.30.0 network 200.10.10.0 network 200.20.20.0 network 200.30.30.0
Output show ip route · Router HQ
R 192.168.5.0/24 [120/1] via 200.30.30.2 C 192.168.10.0/24 directly connected R 192.168.11.0/24 [120/1] via 200.10.10.2 R 192.168.12.0/24 [120/1] via 200.20.20.2 R 192.168.21.0/24 [120/1] via 200.10.10.2 R 192.168.31.0/24 [120/1] via 200.10.10.2 R 200.40.40.0 [120/1] via 200.10.10.2 via 200.30.30.2
✦ Supporto VLSM con subnet mask classless
✦ no auto-summary per evitare summarizzazione automatica
✦ Convergenza verificata su ogni router

06 · VPN site-to-site

07 · Sicurezza perimetrale

Firewall ACL & DMZ fisica

In sostituzione del firewall ASA, la sicurezza perimetrale è implementata tramite ACL estese sulle interfacce WAN. L'ACL permette solo traffico autorizzato in ingresso.

UDP 520 Traffico RIP v2
UDP 53 Query DNS
ICMP Ping diagnostico
TCP 80 Traffico HTTP
TCP 443 Traffico HTTPS
DENY ALL Tutto il resto bloccato
Configurazione ACL firewall
access-list 100 permit udp any any eq 520 access-list 100 permit udp any any eq 53 access-list 100 permit icmp any any access-list 100 permit tcp any any eq 80 access-list 100 permit tcp any any eq 443 access-list 100 deny ip any any ! Applicata su interfacce WAN: interface Serial2/0 ip access-group 100 in

DMZ fisica · Data Center

Il Server Web (192.168.100.10) è collegato direttamente a una porta dedicata del Router DC (FastEthernet1/0), separata fisicamente dalla rete interna.

Interfaccia Router DCReteIPRuolo
FastEthernet0/0LAN interna192.168.5.1/24Server DHCP, DNS, DB
FastEthernet1/0DMZ192.168.100.1/24Server Web isolato
ACL zona DMZ
ip access-list extended ACL-DMZ permit tcp any host 192.168.100.10 eq 80 permit tcp any host 192.168.100.10 eq 443 permit tcp any host 192.168.100.10 established deny ip host 192.168.100.10 192.168.0.0 0.0.255.255 permit ip any any interface FastEthernet1/0 ip access-group ACL-DMZ in
